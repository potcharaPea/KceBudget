## ข้อมูลพื้นฐาน

**Template ปัจจุบัน:** `ใบตัดงบ_template_final.docx`
Token ที่มีอยู่แล้ว: `{CHK_VEH}`, `{CHK_TRV}`, `{CHK_CRN}`, `{CRN_NAME}`, `{CHK_CRT}`, `{CRT_NAME}`, `{CHK_DLY}`, `{DLY_DATE}`, `{DLY_TEAM}`, `{CHK_CON}`, `{CON_NO}`, `{CON_DATE}`, `{CHK_OIL}`, `{CHK_OTH}`, `{WBS}`, `{DPT}`, `{NET}`, `{CAT}`, `{ACT}`, `{YB}`, `{YS}`, `{JB}`, `{JS}`, `{KB}`, `{KS}`, `{CB}`, `{CS}`, `{LB}`, `{LS}`, `{JOB}`, `{USR}`, `{POS}`, `{DD}`, `{MM}`, `{YY}`, `{REF}`, `{APV}`, `{APD}`

**Token ใหม่ที่ต้องเพิ่มใน template (patch 2):**
- `{NODE}` — เลขโหนดท้าย WBS เช่น `02.2`, `01.1` (ถ้าไม่มีให้เป็นว่าง)
- `{OPER}` — ผู้ดำเนินการ: "กฟภ." หรือชื่อบริษัท
- `{DLY_FROM}` / `{DLY_TO}` — วันที่เริ่ม/สิ้นสุด ค่าแรงคนงานรายวัน (แทน `{DLY_DATE}` เดิม)
- `{CRN_FROM}` / `{CRN_TO}` — วันที่ พขร. เครนสว่านเจาะ
- `{CRT_FROM}` / `{CRT_TO}` — วันที่ พขร. รถกระเช้า

**WBS parser — เพิ่ม pattern ใหม่:**
ตัวอย่างเก่า: `I-69-E-KCE69.M4.1104` → WBS เต็ม, ไม่มี node
ตัวอย่างใหม่: `C-68-E-KCECS.0080.02.2` → WBS เต็มทั้งก้อน (รวม `.02.2`), node = `02.2`

กฎ detect node: ถ้า WBS ลงท้ายด้วย pattern `\.\d{2}\.\d$` (เช่น `.02.2`, `.01.1`, `.03.1`) → node = 2 segment สุดท้าย
รูปแบบ WBS ที่ parser ต้องรองรับ (ไม่ตายตัวแค่ `I-69`):
```regex
^[A-Z]-\d{2}-[A-Z]-[A-Z0-9]+\.[A-Z0-9.]+$
```
Parser ต้องอ่าน WBS จาก section 3 ของ ZPSR018 (แถวที่มี WBS + กฎการชำระบัญชี)
และจาก footer บรรทัดสุดท้าย (บรรทัด "เรื่อง ขอนำส่ง..." มักมี WBS กำกับไว้)

## ข้อ 1 — เลขรันแฟ้ม KCE01, KCE02...

**Logic:**
- 1 WBS = 1 แฟ้ม → รหัสแฟ้มรันตามลำดับที่ import เข้าระบบ
- Format: `KCE` + เลขรัน 2 หลัก zero-pad → `KCE01`, `KCE02`, ..., `KCE99`
- บันทึกลง Google Sheet แท็บ `งบ` คอลัมน์ `รหัสแฟ้ม`
- โชว์รหัสแฟ้มบน card แต่ละ WBS และในหน้ารายละเอียด
- **ค้นหาได้:** เพิ่ม search box กรองตามรหัสแฟ้มหรือ WBS ในหน้า "เลือกแฟ้มงาน"
- ถ้า import WBS เดิมซ้ำ (re-import) → ใช้รหัสแฟ้มเดิม ไม่รันใหม่

## ข้อ 2 — วันที่ 2 ช่อง (from - to) สำหรับค่าแรง

**ใช้กับ 3 หัวข้อ:**
1. ค่าแรง พขร.(บ.) เครนสว่านเจาะ → `{CRN_FROM}` - `{CRN_TO}`
2. ค่าแรง พขร.(บ.) รถกระเช้า → `{CRT_FROM}` - `{CRT_TO}`
3. ค่าแรงคนงานรายวัน → `{DLY_FROM}` - `{DLY_TO}`

**แก้ฟอร์มเปิดใบตัดงบ:**
- ทั้ง 3 หัวข้อ เปลี่ยนจากช่องวันที่เดิมเป็น **2 date picker**: วันที่เริ่ม → วันที่สิ้นสุด
- format: `dd/mm/yyyy - dd/mm/yyyy` (แสดงในใบตัดงบ)
- แสดงช่องวันที่เฉพาะเมื่อ checkbox ของหัวข้อนั้นถูกติ๊ก

**แก้ template:**
- `{DLY_DATE}` เดิม → แทนที่ด้วย `{DLY_FROM}-{DLY_TO}` ใน paragraph ค่าแรงรายวัน
- เพิ่ม `{CRN_FROM}-{CRN_TO}` ใน paragraph ค่าแรง พขร.เครน (หลัง `{CRN_NAME}`)
- เพิ่ม `{CRT_FROM}-{CRT_TO}` ใน paragraph ค่าแรง พขร.กระเช้า (หลัง `{CRT_NAME}`)
- บันทึกเป็น `ใบตัดงบ_template_v_final2.docx`

## ข้อ 3 — แก้ WBS parser + กรอกมือได้

**ปัญหา:** Parser เดิมอ่าน WBS ได้แค่บาง pattern เช่น `I-69-E-...` แต่อ่าน `C-68-E-KCECS.0080.02.2` ไม่ได้

**แก้ parser:**
- ใช้ regex กว้างขึ้น: `^[A-Z]-\d{2}-[A-Z]-[A-Z0-9]+\.[A-Z0-9.]+` ครอบคลุมทุก WBS format
- อ่านจากทั้ง section 3 ของ ZPSR018 และ footer บรรทัดสุดท้าย
- วน loop ทุกหน้าของ PDF (กรณีไฟล์หลายหน้า)

**กรณี parser อ่านไม่ได้:**
- popup แจ้ง "อ่าน WBS ไม่ได้" พร้อมช่องให้กรอก WBS มือ
- **validation:** ถ้าไม่กรอก WBS → ไม่สามารถ import งานได้ (block ปุ่ม confirm)
- WBS ที่กรอกมือบันทึกลง Sheet เหมือนกัน

**แก้ไข WBS ภายหลัง:**
- ปุ่ม edit บน card แฟ้ม → แก้ WBS ได้ (กรณีระบบอ่านผิด)
- บันทึกประวัติการแก้ไขลง Sheet แท็บ `ประวัติแก้งบ` (WBS เก่า → ใหม่, วันที่, ผู้แก้)

## ข้อ 4 — แสดง Node ใน template และแอป

**Logic detect node จาก WBS:**
```javascript
function extractNode(wbs) {
    // match xx.x ท้าย WBS เช่น .02.2, .01.1, .03.1
    const match = wbs.match(/\.(\d{2}\.\d)$/);
    return match ? match[1] : '';   // '' ถ้าไม่มี node
}
// ตัวอย่าง:
// 'C-68-E-KCECS.0080.02.2' → '02.2'
// 'I-69-E-KCE69.M4.1104'   → ''
```

**แสดงในแอป:**
- card แฟ้ม: ถ้ามี node แสดงแท็ก "โหนด: 02.2" ใต้ WBS
- หน้ารายละเอียดแฟ้ม: แสดงทั้ง WBS เต็มและ node แยก field

**แก้ template:**
- เพิ่ม token `{NODE}` ใน row4 c1 ถัดจาก `หมายเลขงาน {WBS}`
  → `หมายเลขงาน  {WBS}` (บรรทัดเดิม)
  → เพิ่มบรรทัดใหม่: `โหนด  {NODE}` (ถ้า node ว่างให้ GAS replaceText ด้วย `''`)
- บันทึกใน `ใบตัดงบ_template_v_final2.docx`

## ข้อ 6 — หน้าค้นหางบที่เหลือ

**เพิ่มหน้า/section ใหม่ในแอป** (เมนู sidebar: "ค้นหางบที่เหลือ")

**Filter ที่ต้องมี:**
1. **หมวดงบ** (multi-select): พัสดุเข้างาน, ค่าแรงงาน/จ้างเหมา, ค่าควบคุมงาน, ค่าขนส่ง, ค่าเบ็ดเตล็ด, ค่าดำเนินการ
2. **ผู้ดำเนินการ**: กฟภ. / บริษัท / ทั้งหมด
3. **เงื่อนไขบังคับ (AND):**
   - ยอดคงเหลือหมวด > 0
   - ยอดงบจัดสรรรวม (ของ WBS นั้น) > 0

**ผลลัพธ์แสดงเป็นตาราง:**

| รหัสแฟ้ม | WBS | โหนด | แผนก | หมวดงบ | ยอดจัดสรร | จ่ายแล้ว | คงเหลือ | ผู้ดำเนินการ |
|---|---|---|---|---|---|---|---|---|

- Sort ได้ตามคอลัมน์
- กดที่แถวเพื่อเปิดใบตัดงบของหมวดนั้นได้เลย
